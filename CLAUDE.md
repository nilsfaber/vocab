# Claude instructions

## Three-pillar architecture

This project has three distinct, independent concerns:

| Pillar | Folder | Purpose | Where it runs |
|--------|--------|---------|---------------|
| **Vocab extraction** | root (`extract.py`) | Pull words from KOReader / local sqlite, enrich with definitions, write `docs/vocab.json` | Dev machine |
| **Image generation** | `imagegen/` | Ollama + ComfyUI pipeline to generate and manage per-word images | Dev machine (GPU) |
| **PWA** | `docs/` | Deployable web app served on GitHub Pages | Browser / GitHub Pages |

**`docs/vocab.json` is the unified database** shared by all three pillars. It is object-keyed by lowercase word and contains definitions, translations, synonyms/antonyms, image metadata, default image, and regen flags. `extract.py` adds new words and enriches them; `imagegen/` reads and writes image metadata; the PWA reads it to display cards and images.

## PWA (`docs/`)

After any change to `docs/index.html`, `docs/sw.js`, or `docs/manifest.json`:
- Bump the cache name in `docs/sw.js` (e.g. `vocab-v1` → `vocab-v2`).
- If new static files were added, add them to the `SHELL` array in `docs/sw.js`.

Never commit `docs/vocab.json` or anything under `docs/images/`.

## Vocab extraction (`extract.py`)

Run from repo root. Reads from device via adb or local `data/vocab.sqlite3`.
Writes enriched vocab to `docs/vocab.json`.

Adding a new word manually:
1. If it exists in KOReader, just re-run `python extract.py`.
2. For a word not in KOReader, add it to `data/manual_words.json` (list of word strings). `extract.py` will fetch its definition and include it.

## Image generation (`imagegen/`)

### Starting the dev server

Always use `server.py` instead of `python -m http.server`. It serves both UIs **and** provides the API for running scripts.

```bash
# From repo root:
python server.py
```

Then open:
- `http://localhost:8765/` → imagegen management UI
- `http://localhost:8765/docs/` → docs PWA
- `http://localhost:8188/` → ComfyUI (started separately)

### Prerequisites

- `ollama serve` running with `gemma4:e2b` pulled
- `python ComfyUI/main.py` running
- `pngquant` installed

### Generation scripts

Primary script: `imagegen/generators/3_gemma4b_sdxl_refiner.py`. Each script:
- Phase 1: Ollama generates a scene brief + SDXL keywords → cached to `imagegen/scenes.json`
- Phase 2: ComfyUI renders image via SDXL base + refiner
- Saves full-res PNG to `imagegen/words/<word>/DDMMYY_N_<script>.png`
- Writes image metadata directly into `docs/vocab.json` (the unified db)

CLI flags:
```
--words <word> [<word> ...]   # process only these words (implies --force)
--force                       # regenerate even if image exists
--limit N                     # cap number of words processed
```

### imagegen UI behaviour

Both imagegen and the PWA share the same bottom bar design: a **glass context row** (changes per mode) above a **glass pill** (game ⚡ | gallery ⊞ | word 📖) plus a **UI toggle** button.

- **Word list**: all words from vocab.json shown. Red dot = no default set; orange dot = flagged for regen; blue dot = has new unseen images.
- **Navigation**: swipe left/right anywhere in the main area, or press ← → arrow keys when a word is open. No visible prev/next buttons.
- **Unsaved changes guard**: navigating away while editing shows a modal with Save / Discard / Cancel.
- **Context row**: shows live score when in game mode (with mode selector); columns slider in gallery mode; queue indicator (jobs running/queued) in word mode.
- **Game modes**: Random (default, picks a viable mode each round), Context (definition/sentence → pick word), Definitions (word → pick definition), Image (image → pick word), Dutch (translation → pick word). Pool filtered per mode; stats shared across modes per word.
- **Word of the Day** (PWA only): on app open, if >1 hour since last shown, displays a modal with the day's word, part of speech, and definition. Word is deterministic per calendar date.
- **Selecting an alternative image** automatically applies it as the default AND writes it to `docs/images/<word>.png` (via pngquant if available). default_image is persisted in `docs/vocab.json` via the server API.
- **Word info panel** — single glass card split by a vertical divider. Left side: word title, delete button, pencil/save toggle, definition textarea. Right side: inline rows for Dutch translation, Synonyms, Antonyms. The pencil button in the title bar toggles edit mode for all fields at once (icon switches pencil↔check); all fields save in one API call on confirm.
- **Flag button** (⚑) in word title bar (imagegen only) — toggles regen flag in vocab.json.
- **Approved toggle** (✓) on each alt thumbnail — marks the image as approved in vocab.json.
- **Run panel** — select script, optionally edit the prompt, click Run. Output streams live. On completion vocab reloads and the new image appears without a full page reload.
- **Page state** persisted in `localStorage` (`imagen_open_word`).
- **Icons**: SVG sprite at `docs/icons.svg`; imagegen references as `/docs/icons.svg#icon-*`, PWA as `icons.svg#icon-*`.
- **Mobile (≤ 600px)**: sidebar collapses to a fixed bottom search bar above the panel controls. The word list appears as an upward overlay when the search field is focused. Panel controls span full screen width.

### vocab.json schema (per word)

```json
{
  "word": "alacrity",
  "part_of_speech": "noun",
  "definition": "...",
  "example": "...",
  "translation": { "nl": "bereidwilligheid" },
  "synonyms": [], "antonyms": [],
  "enriched": true,
  "occurrences": [...],
  "images": [
    { "filename": "...", "script": "...", "prompt_base": "...", "scene": "...", "date": "...", "approved": false }
  ],
  "default_image": "filename.png",
  "flagged_for_regen": false,
  "stats": { "correct": 0, "wrong": 0 }
}
```

### Server API (`server.py`)

- `GET /api/vocab` — returns full `docs/vocab.json`
- `POST /api/update-word` — body: `{word, ...fields}` — updates any fields for a word
- `POST /api/add-word` — body: `{word, ...}` — inserts a new word with defaults
- `DELETE /api/word` — body: `{word}` — removes a word from vocab.json
- `POST /api/apply` — body: `{word, filename}` — runs pngquant, writes to `docs/images/`, sets `default_image`
- `GET /api/scripts` — lists generation scripts
- `POST /api/run` — enqueues a generation job (serial queue)
- `GET /api/status/<id>` — polls job output
- `GET /api/queue` — returns `{running, pending}` job IDs

### Writing images to docs

Selecting an alternative in the UI calls `/api/apply` which runs pngquant on the hires source and writes `docs/images/<word>.png`, and sets `default_image` in vocab.json.

## General rules

- `data/` is gitignored — never commit it.
- `docs/vocab.json` and `docs/images/` are gitignored — never commit them.
- `imagegen/scenes.json` and `imagegen/words/` are gitignored.
- `server.py` (repo root), `imagegen/index.html`, `docs/vocab.css`, `docs/vocab.js`, and `imagegen/generators/*.py` are tracked.
- Python scripts run from the **repo root**, not from their own folder.
- Primary generation script is `imagegen/generators/3_gemma4b_sdxl_refiner.py`.
- `docs/vocab.css` and `docs/vocab.js` are the shared UI layer — imagegen references them as `/docs/vocab.css` (absolute path via server.py) and the PWA references them as `vocab.css` (relative).
