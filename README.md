# Vocabulary Builder

Extract vocabulary from a KOReader device, enrich with definitions and images, and review in a PWA.

## Three pillars

| | Folder | Runs on |
|---|---|---|
| **Vocab extraction** | `extract.py` (repo root) | Dev machine |
| **Image generation** | `imagegen/` | Dev machine (GPU) |
| **Web app** | `docs/` | Browser / GitHub Pages |

## Pillar 1 — Vocab extraction

### From a KOReader device

1. Connect Android device via USB with USB debugging enabled.
2. Run from the repo root:

```bash
python extract.py
```

This:
- Pulls `vocabulary_builder.sqlite3` → `data/vocab.sqlite3`
- Pulls all `.epub` files from the device → `data/books/`
- Parses epubs for in-context sentence snippets per word
- Exports enriched vocab → `docs/vocab.json` + `docs/vocab_public.json`
- Fetches definitions (phonetic, part of speech, all meanings, synonyms/antonyms) from [dictionaryapi.dev](https://dictionaryapi.dev/)
- Fetches Dutch translations from MyMemory

Re-running is safe — existing enriched fields and cached epubs are preserved.

### Hardcover integration

Set `HARDCOVER_TOKEN` in your environment to filter epub scanning to books you have actually read:

```bash
export HARDCOVER_TOKEN=your_token_here
python extract.py
```

With a valid token, `extract.py` queries Hardcover for your read list, matches local epubs, and collects all occurrences (no cap). Without a token it scans all local epubs and caps at 10 occurrences per word.

At startup the script reports: `✅ Hardcover connected — N books marked as read` or `⚠ No HARDCOVER_TOKEN — scanning all local books`.

### Adding a word manually

Add the word string to `data/manual_words.json` (create if missing):
```json
["ephemeral", "sanguine"]
```
Then re-run `python extract.py`. Definitions are fetched automatically.

### vocab.json schema (per word)

```json
{
  "word": "alacrity",
  "phonetic": "/əˈlækrɪti/",
  "part_of_speech": "noun",
  "definition": "...",
  "example": "...",
  "definitions": [{ "part_of_speech": "noun", "definition": "...", "example": "..." }],
  "translation": { "nl": "bereidwilligheid" },
  "synonyms": [], "antonyms": [],
  "enriched": true,
  "learnt": false,
  "deleted": false,
  "occurrences": [{ "book": "...", "paragraph": "single sentence containing the word" }],
  "images": [{ "filename": "...", "script": "...", "prompt_base": "...", "scene": "...", "date": "..." }],
  "default_image": "filename.png",
  "flagged_for_regen": false,
  "stats": {
    "context":  { "correct": 0, "wrong": 0 },
    "reverse":  { "correct": 0, "wrong": 0 },
    "image":    { "correct": 0, "wrong": 0 },
    "dutch":    { "correct": 0, "wrong": 0 },
    "total":    { "correct": 0, "wrong": 0 }
  }
}
```

Deleted words remain in `vocab.json` as `"deleted": true` so they are not re-imported on the next run.

## Pillar 2 — Image generation

### Prerequisites

```bash
source venv/bin/activate
ollama serve               # keep running
python ComfyUI/main.py     # keep running
```

### Start the dev server

`server.py` serves the imagegen UI, the docs PWA, and the generation API — all in one process.

```bash
python server.py
```

Open:
- `http://localhost:8765/` → imagegen management UI
- `http://localhost:8765/docs/` → docs PWA
- `http://localhost:8188/` → ComfyUI (separate process)

### Run a script from the command line

```bash
python imagegen/generators/3_gemma4b_sdxl_refiner.py           # all words
python imagegen/generators/3_gemma4b_sdxl_refiner.py --words brethren alacrity
python imagegen/generators/3_gemma4b_sdxl_refiner.py --force   # regenerate
python imagegen/generators/3_gemma4b_sdxl_refiner.py --limit 5
```

### imagegen UI

Both imagegen and the PWA share the same bottom bar: a **glass context row** (pill-shaped, mode-specific) above a **glass pill** (game ⚡ | gallery ⊞ | word 📖) and a **UI-toggle** button (circle outline = UI visible; filled dot = UI hidden — hides the context row and pill).

**Gallery mode**
- Sortable grid (A–Z, date added, occurrences, score). Cycling columns button (1 → 2 → 3 → 4).
- Dots: 🔴 no default image, 🟠 flagged for regen, 🔵 new unseen images.
- Context row: ⚙ Settings, ＋ Add word, columns button.

**Word mode**
- Navigate with swipe left/right, ← → arrow keys, or the ← → buttons in the context row.
- Word title bar: phonetic pronunciation, learnt toggle (⭐), flag button (⚑, imagegen only), delete button.
- Words with no image show a placeholder at the standard card aspect ratio.
- **Word info panel** — glass card split by a vertical divider. Left: definition textarea. Right: Dutch translation, synonyms, antonyms as inline rows. All fields auto-save 600 ms after the last keystroke (no save button needed).
- **Definitions list** — full dictionary entry (all parts of speech, examples) rendered below the editable field.
- **Per-mode game stats** — progress bars (correct %) per game mode shown at the bottom of the word panel.
- **Occurrences** — grouped by book (collapsible), each showing the exact sentence containing the word.
- **Alternatives** — thumbnail grid of all generated images; click to set as default (runs pngquant → `docs/images/<word>.png`).
- **Run panel** — select a generation script, edit prompts, click **▶ Run**. Jobs queue; output streams live.
- **Flag button** (⚑) — marks a word for regeneration.

**Game mode**
- Five modes selectable in the context row: **Random** (default), **Context** (sentence → word), **Definitions** (word → definition), **Image** (image → word), **Dutch** (Dutch → word).
- Four-choice answers; stats saved per word per mode.
- Quick-add bar slides in below the game card without leaving game mode.

**Settings modal** (⚙ in gallery context row)
- Theme selector: **Dark** / **Light** / **E-paper** (persists in localStorage). E-paper uses pure white backgrounds, no blur effects, and zero animation durations — optimised for e-ink screens.
- Export / Import vocabulary (JSON).
- Reset game stats.

### Image file layout

```
imagegen/
└── words/
    └── <word>/
        └── DDMMYY_N_<scriptname>.png
```

### Server API

| Endpoint | Method | Purpose |
|---|---|---|
| `/api/vocab` | GET | Full `docs/vocab.json` |
| `/api/update-word` | POST | Update any fields for a word |
| `/api/add-word` | POST | Insert a new word with defaults |
| `/api/word` | DELETE | Soft-delete a word (`deleted: true`) |
| `/api/apply` | POST | pngquant + write `docs/images/<word>.png` |
| `/api/scripts` | GET | List available generation scripts |
| `/api/run` | POST | Enqueue a generation job |
| `/api/status/<id>` | GET | Poll job output |
| `/api/queue` | GET | Running + pending jobs |

Every write also updates `docs/vocab_public.json` (occurrence paragraphs stripped).

## Pillar 3 — Web app (`docs/`)

Served via `server.py` at `http://localhost:8765/docs/`, or deployed to GitHub Pages.

### Features

- **Gallery** — sortable grid (A–Z, date added, occurrences, score). Cycling columns button (1–4) in context row.
- **Word view** — default image (placeholder shown when none), definition, full dictionary entry, Dutch translation, synonyms, antonyms, per-mode stats, occurrences. All fields auto-save. Navigate with swipe, ← → keys, or the prev/next buttons in the context row.
- **Game mode** — five modes (Random, Context, Definitions, Image, Dutch) with weighted word selection and per-mode stats.
- **Word of the Day** — on open, if >1 hour since last shown, a modal displays the day's word with part of speech and definition. Deterministic per calendar date.
- **Settings** (⚙ in gallery context row) — theme (Dark / Light / E-paper), sync vocab, export vocab, reset stats.
- **Add / delete word** — ＋ in gallery context row; delete in word title bar with confirmation.
- **Learnt toggle** — star icon in word title bar; also available as an action after a correct game answer.
- **Soft delete** — deleted words stay in `vocab.json` as `deleted: true`; they are excluded from all views and not re-imported.
- **Mobile (≤ 600px)** — sidebar collapses to a search bar above the panel controls; word list appears as an upward overlay when searching.
- **Offline** — app shell and `vocab.json` cached by service worker. Network-first for `vocab.json`.

### PWA / offline

Installable via "Add to Home Screen". The service worker caches the app shell (relative paths for GitHub Pages subpath compatibility) and serves `vocab.json` network-first with a cache fallback. Navigation requests always resolve to the cached `index.html`.

## Shared CSS/JS (`docs/vocab.css`, `docs/vocab.js`)

Both `imagegen/index.html` and `docs/index.html` use these files. Imagegen references them as `/docs/vocab.css` (absolute URL via server.py); the PWA references them as `vocab.css` (relative).

Overridable hooks in `vocab.js`:
- `VOCAB_SERVER` — set to the API base URL; enables server persistence (imagegen sets this).
- `ICON_BASE` — prefix for SVG sprite path; imagegen sets to `/docs/`, PWA uses `''`.
- `imgPath(word, filename)` — defaults to `images/<word>.png` (PWA); imagegen overrides to `words/<word>/<filename>`.
- `loadVocab()` — imagegen overrides to always fetch from `VOCAB_SERVER/api/vocab`.

## Dictionary data

Definitions from [dictionaryapi.dev](https://dictionaryapi.dev/) — free, no API key. English only. Unfound words are marked `enriched: true` with empty fields and can be filled manually via the word info panel or `data/manual_words.json`.
